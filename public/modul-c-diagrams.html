5Public Abnahme Sign — Sequence

Kunde public link ile sayfayı açar. Restbetrag canlı hesaplanır, gönderilmiş Rechnung'lar listelenir.

sequenceDiagram
  participant Admin
  participant FE as Frontend
  participant BE as Backend
  participant DB
  participant Kunde

  Admin->>FE: Status → abnahme + Speichern
  FE->>BE: POST /forms/:id/abnahme/sign-request
  BE->>DB: INSERT sign_request (token_hash, snapshot_json)
  BE-->>FE: signUrl
  FE->>Admin: EmailComposer + Restbetrag satırı

  Note over Kunde: Mail link tıklar
  Kunde->>BE: GET /public/abnahme-sign/:token
  BE->>DB: SELECT sign_request
  BE->>DB: SELECT angebot.brutto + SUM(anzahlungen)
  BE->>DB: SELECT rechnungen WHERE status='gesendet'
  BE-->>Kunde: snapshot + restbetrag + rechnungen[]

  Note over Kunde: Sayfa render: Zahlungsübersicht + PDF link'leri
  Kunde->>BE: GET /public/abnahme-sign/:token/rechnung/:id/pdf
  BE-->>Kunde: PDF blob (form_id sahiplik kontrolü)

  Kunde->>BE: POST sign (signature_data + name)
  BE->>DB: UPDATE sign_request SET status=signed
  BE->>DB: UPDATE form SET status=abnahme
  BE-->>Kunde: Vielen Dank

6Email Send Sequence (rechnung_id)

EmailComposer "Jetzt senden" tıklayınca arka planda neler oluyor.

sequenceDiagram
  participant Admin
  participant FE as Frontend
  participant BE as Backend
  participant DB
  participant SMTP

  Admin->>FE: Send tıkla
  FE->>BE: POST /email/send {rechnung_id}
  BE->>DB: SELECT rechnung WHERE id=$1
  DB-->>BE: row + branch_id + generated_pdf
  BE->>BE: branch_id check
  BE->>BE: attach generated_pdf blob
  BE->>SMTP: transporter.sendMail()
  SMTP-->>BE: delivered

  alt rechnung.status='entwurf'
    BE->>DB: UPDATE rechnung SET status='gesendet', sent_at=NOW()
    BE->>DB: UPDATE form SET status='gesendet' or 'rest_rechnung_erstellt'
  end

  BE->>DB: INSERT email_log
  BE-->>FE: 200 OK
  FE-->>Admin: Toast E-Mail gesendet

7DB Schema — Modul C

3 yeni tablo + ilişkiler. items_json snapshot olarak immutable.

erDiagram
  aufmass_forms ||--o{ aufmass_rechnungen : has
  aufmass_forms ||--o{ aufmass_anzahlungen : has
  aufmass_rechnungen ||--o| aufmass_anzahlungen : "auto-linked (1:1)"
  aufmass_forms ||--o{ aufmass_angebot : has
  aufmass_angebot ||--o{ aufmass_angebot_items : has
  aufmass_users ||--o{ aufmass_rechnungen : created_by

  aufmass_rechnungen {
    serial id PK
    varchar rechnung_nr UK
    varchar type "anzahlungs|schluss"
    int form_id FK
    varchar branch_id
    varchar kunde_vorname "snapshot"
    varchar kunde_nachname "snapshot"
    decimal netto_betrag
    decimal mwst_satz
    decimal mwst_betrag
    decimal brutto_betrag
    date rechnungsdatum
    date leistungsdatum
    date zahlungsziel
    jsonb items_json "snapshot"
    varchar status "entwurf|gesendet"
    timestamp sent_at
    timestamp paid_at
    bytea generated_pdf
    timestamp created_at
  }

  aufmass_anzahlungen {
    serial id PK
    int form_id FK
    int rechnung_id FK "auto"
    decimal betrag
    date zahlungsdatum
    varchar zahlungsmethode "default überweisung"
    int beleg_file_id FK
    text notiz
    varchar branch_id
    timestamp created_at
  }

  aufmass_rechnung_counters {
    varchar branch_key PK
    int year PK
    int last_num "race-safe UPSERT"
  }
Format: KOB-R-2026-00001 (branchKey + R + year + 5-haneli sıra)
