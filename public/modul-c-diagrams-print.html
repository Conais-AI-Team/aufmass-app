Public Sign Sequence

Sequence · 7 / 11

Müşterinin Abnahme imzası için açtığı public link. Restbetrag canlı hesaplanır, gönderilmiş Rechnung'lar PDF olarak listelenir.

sequenceDiagram
  participant Admin
  participant FE as Frontend
  participant BE as Backend
  participant DB
  participant Kunde

  Admin->>FE: Status → abnahme + Speichern
  FE->>BE: POST /forms/:id/abnahme/sign-request
  BE->>DB: INSERT sign_request
  BE-->>FE: signUrl
  FE->>Admin: Email + Restbetrag

  Note over Kunde: Mail link tıklar
  Kunde->>BE: GET /public/abnahme-sign/:token
  BE->>DB: SELECT angebot + sum(anzahlungen)
  BE->>DB: SELECT rechnungen gesendet
  BE-->>Kunde: snapshot + restbetrag + rechnungen

  Kunde->>BE: GET /public/.../rechnung/:id/pdf
  BE-->>Kunde: PDF blob (form_id check)

  Kunde->>BE: POST sign + signature
  BE->>DB: UPDATE form abnahme + sign signed
  BE-->>Kunde: Vielen Dank

Email Send Sequence (rechnung_id)

Sequence · 8 / 11

EmailComposer Send tıklayınca backend rechnung'u branch-check eder, PDF'i ek olarak alır, SMTP'ye gönderir, başarı sonrası rechnung+form status'unu atomik gesendet'e geçirir.

sequenceDiagram
  participant FE as Frontend
  participant BE as Backend
  participant DB
  participant SMTP

  FE->>BE: POST /email/send {rechnung_id}
  BE->>DB: SELECT rechnung WHERE id=$1
  DB-->>BE: row + branch + generated_pdf
  BE->>BE: branch_id check
  BE->>BE: attach PDF blob
  BE->>SMTP: sendMail()
  SMTP-->>BE: delivered

  alt rechnung.status='entwurf'
    BE->>DB: UPDATE rechnung gesendet, sent_at
    BE->>DB: UPDATE form gesendet/rest_rechnung_erstellt
  end

  BE->>DB: INSERT email_log
  BE-->>FE: 200 OK

DB Schema — Modul C Tables

ER · 9 / 11

3 yeni tablo. items_json snapshot olarak immutable — Rechnung kesildikten sonra Angebot değişse de fatura içeriği değişmez (yasal). Counter race-safe UPSERT.

erDiagram
  aufmass_forms ||--o{ aufmass_rechnungen : has
  aufmass_forms ||--o{ aufmass_anzahlungen : has
  aufmass_rechnungen ||--o| aufmass_anzahlungen : "1:1 auto-link"

  aufmass_rechnungen {
    serial id PK
    varchar rechnung_nr UK "KOB-R-2026-00001"
    varchar type "anzahlungs schluss"
    int form_id FK
    varchar branch_id
    decimal netto_betrag
    decimal mwst_satz
    decimal mwst_betrag
    decimal brutto_betrag
    date rechnungsdatum
    date zahlungsziel
    jsonb items_json "snapshot"
    varchar status "entwurf gesendet"
    timestamp sent_at
    bytea generated_pdf
  }

  aufmass_anzahlungen {
    serial id PK
    int form_id FK
    int rechnung_id FK "auto"
    decimal betrag
    date zahlungsdatum
    int beleg_file_id
    text notiz
  }

  aufmass_rechnung_counters {
    varchar branch_key PK
    int year PK
    int last_num "race-safe"
  }
